=== Invoice subtype codes Saudi Arabia

In addition to the invoice type code it is required to provide invoice subtypes. 

This should be placed in `cbc:InvoiceTypeCode/@name` and must follow a certain syntax.

Structure of the value must be *NNPNESB* (e.g. `0100000`) where:

|===
|Value |Supported values


|`NN (Position 1 and 2)`
|01 for tax invoice +
02 for simplified tax invoice.

|`P (Position 3)`
|3rd party invoice transaction, 0 for false, 1 for true.

|`N (Position 4)`
|Nominal invoice transaction, 0 for false, 1 for true.

|`E (Position 5)`
|Exports invoice transaction, 0 for false, 1 for true.

|`S (Position 6)`
|Summary invoice transaction, 0 for false, 1 for true.

|`B (Position 7)`
|Self-billed invoice transaction, 0 for false, 1 for true. +
Note that only 0 is currently supported.

=== Business process codes for France

The business process type (Cadre de facturation) is specified using the `@name` attribute on the Invoice Type Code and indicates the nature of the transaction, affecting VAT treatment. The following tables list the currently supported codes.

==== Standard invoice types (Invoice Type Code `380`)

The series indicates the transaction category: B (goods), S (services), M (mixed goods and services).

[cols="1,3,4", options="header"]
|===
|`@name` |Description |When to use

|`B1`
|Goods invoice
|Standard invoice for the sale of goods

|`S1`
|Services invoice
|Standard invoice for the provision of services

|`M1`
|Mixed invoice (goods + services)
|Invoice covering both goods and services that are not incidental to each other

|`B2`
|Goods invoice (already paid)
|Invoice for goods where payment was made before invoicing

|`S2`
|Services invoice (already paid)
|Invoice for services where payment was made before invoicing

|`M2`
|Mixed invoice (already paid)
|Mixed invoice where payment was made before invoicing

|`B4`
|Final invoice for goods (after down payment)
|Final goods invoice issued after one or more prepayment invoices (type `386`)

|`S4`
|Final invoice for services (after down payment)
|Final services invoice issued after one or more prepayment invoices (type `386`)

|`M4`
|Double final invoice (after down payment)
|Final mixed invoice issued after one or more prepayment invoices (type `386`)

|`S5`
|Subcontractor services invoice
|Invoice from a subcontractor for services rendered under a subcontracting arrangement

|`S6`
|Co-contractor services invoice
|Invoice from a co-contractor for services rendered under a co-contracting arrangement

|`B7`
|Goods invoice (e-reporting VAT collected)
|Invoice for goods where VAT was already collected and reported via e-reporting

|`S7`
|Services invoice (e-reporting VAT collected)
|Invoice for services where VAT was already collected and reported via e-reporting

|===

TIP: *Prepayment flow:* A prepayment invoice (type `386`) uses `B1`, `S1`, or `M1` as business process code. The corresponding final invoice (type `380`) then uses `B4`, `S4`, or `M4` and must reference the prepayment invoice via `cac:BillingReference`.

==== Derived document types

The following document types do not define their own business process codes — the `@name` value is derived from the underlying transaction.

[cols="1,3,1,3", options="header"]
|===
|Invoice Type Code |Document Type |UBL Document Type |Business Process Code (`@name`)

|`381`
|Credit note
|CreditNote
|Same code as the original invoice being credited

|`384`
|Corrected invoice
|Invoice
|Same code as the invoice being corrected

|`386`
|Prepayment invoice
|Invoice
|`B1`, `S1`, or `M1` matching the goods/services type

|`389`
|Self-billed invoice
|Invoice
|Same code as would apply to a standard invoice (type `380`)

|`261`
|Self-billed credit note
|CreditNote
|Same code as the self-billed invoice being credited

|`471`
|Self-billed corrected invoice
|Invoice
|Same code as the self-billed invoice being corrected

|`500`
|Self-billed prepayment invoice
|Invoice
|`B1`, `S1`, or `M1` matching the goods/services type

|`503`
|Prepayment credit note
|CreditNote
|Same code as the prepayment invoice being credited

|===

For example, a credit note (`381`) crediting a subcontractor services invoice would use `@name="S5"`, while a corrected invoice (`384`) correcting an already-paid mixed invoice would use `@name="M2"`.

NOTE: The French specification also defines factored invoice types (`393`, `396`, `472`) and self-billed factored types (`501`, `502`). These are not currently supported.
